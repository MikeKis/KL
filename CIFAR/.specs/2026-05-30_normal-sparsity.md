# Функция NormalSparsity

- **Дата**: 2026-05-30
- **Статус**: Draft
- **Связанные issue/PR**: (нет)

## Цели

Реализовать Python-функцию `NormalSparsity(q)`, которая для заданной верхней хвостовой вероятности `q` нормального распределения возвращает математическое ожидание величины:

`max(0, min(1, x / v))`,

где `x` — стандартная нормально распределённая случайная величина, а `v` — квантиль, задаваемый условием `P(x > v) = q`.

## Не цели

- Не реализовывать численное моделирование методом Монте-Карло как основной алгоритм.
- Не добавлять CLI, GUI или файловый ввод/вывод.
- Не менять существующую C++-логику `def_sparsity` / `def_saturation`.
- Не вводить обязательную зависимость от SciPy, если достаточно стандартной библиотеки Python.

## Функциональные требования

1. Функция должна принимать ровно один аргумент `q`.
2. Допустимый диапазон `q`: `0 < q <= 0.5`.
3. При значении `q` вне диапазона `(0, 0.5]` функция должна завершаться ошибкой валидации (`ValueError`).
4. Значения `True` и `False` должны отклоняться как недопустимый тип аргумента, несмотря на то что `bool` является подклассом `int` в Python.
5. Под нормальным распределением по умолчанию понимать стандартное нормальное распределение `N(0, 1)`.
6. Квантиль `v` определяется как `v = Phi^-1(1 - q)`, где `Phi` — CDF стандартного нормального распределения.
7. Для `0 < q < 0.5` результат должен вычисляться аналитически, без случайной выборки:

   `q + (phi(0) - phi(v)) / v`,

   где `phi` — PDF стандартного нормального распределения.
8. Для особого значения `q = 0.5` функция должна возвращать `0.5` как непрерывное продолжение формулы при `v -> 0+`.
9. Функция должна возвращать `float`.
10. Имя публичной функции должно быть ровно `NormalSparsity`; snake_case-алиас `normal_sparsity` не добавлять.

## Нефункциональные требования

- Производительность: один вызов должен выполнять O(1) математических операций; без циклов по выборкам и без генерации случайных чисел.
- Память / ресурсы: O(1), без выделения массивов.
- Потокобезопасность / параллелизм: функция не должна менять глобальное состояние и должна быть безопасна для параллельных вызовов.
- Совместимость (API, форматы файлов, версии toolchain): Python 3.8+; использовать `statistics.NormalDist` и не поддерживать версии Python ниже 3.8.
- Наблюдаемость (логи, метрики): логирование не требуется.
- Прочее: численная устойчивость должна быть явно обработана для `q` около нижней открытой границы `0` и в точке `0.5`.
