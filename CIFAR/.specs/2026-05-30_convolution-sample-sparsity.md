# Эмпирическая sparsity по свёрточным выборкам CIFAR

- **Дата**: 2026-05-30
- **Статус**: Draft
- **Связанные issue/PR**: (нет)

## Цели

Реализовать Python-скрипт, который читает фильтры `Workplace/CIFAR_3x3.PCFilters.txt` и изображения `Workplace/CIFAR10.bin`, для каждого фильтра строит случайную выборку `S` из 300000 значений свёртки на случайных плитках случайных изображений и сохраняет в `sparsity.csv` значения функции `spar(S; q, n)` для `q` от `0.0001` до `0.5` с шагом `0.0001`.

## Не цели

- Не менять аналитическую природу `NormalSparsity(q, n)` для стандартного нормального распределения; этот скрипт считает только `spar(S; q, n)` на выборках свёрток.
- Не менять формат файлов `*.PCFilters.txt` и `CIFAR10.bin`.
- Не переписывать существующую C++-консольную свёртку и генерацию PC-фильтров.
- Не строить карты свёрток для всех изображений целиком, если для задачи достаточно случайных плиток.
- Не добавлять визуализацию, графики или интерактивный интерфейс.

## Функциональные требования

1. Скрипт должен запускаться из корня проекта и по умолчанию использовать входные файлы:
   - `Workplace/CIFAR_3x3.PCFilters.txt`;
   - `Workplace/CIFAR10.bin`.
2. Скрипт должен создавать CSV-файл `sparsity.csv` в текущей рабочей директории или в явно заданном выходном пути.
3. Формат фильтров должен соответствовать существующему формату `SavePCFilters` / `LoadPCFilters`: секция `[PCFilters]`, `version=1`, `count=N`, далее секции `[Filter0] ... [FilterN-1]` с `rows`, `cols`, `channels`, `type`, `values`.
4. Формат `CIFAR10.bin` должен соответствовать текущему проектному формату: подряд без заголовка, каждое изображение занимает `32 * 32 * 3` байта `uint8`, порядок каналов interleaved, без target-меток.
5. Скрипт должен проверить, что размер `CIFAR10.bin` кратен `32 * 32 * channels`, где `channels` берётся из фильтров; для `CIFAR_3x3.PCFilters.txt` ожидается `channels=3`.
6. Для каждого фильтра скрипт должен сформировать независимую выборку размера `300000`.
7. Один элемент выборки должен вычисляться как скалярное произведение фильтра и случайной плитки случайного изображения.
8. Случайная плитка выбирается так, чтобы область фильтра полностью лежала внутри изображения: `x in [0, image_width - filter_cols]`, `y in [0, image_height - filter_rows]`.
9. Случайный выбор изображений и координат выполняется с возвращением.
10. Выборки для разных фильтров должны быть независимыми.
11. Для каждого значения `q` из сетки `0.0001, 0.0002, ..., 0.5` скрипт должен вычислить `spar(S; q, n)`, где `S` — выборка свёрточных значений текущего фильтра:
    - найти порог `v(q)` по выборке так, чтобы он соответствовал верхней хвостовой вероятности `q`;
    - вычислить среднее по выборке от `max(0, min(1, floor((x / v(q)) / (1 / n)) * (1 / n)))`.
12. Для `q=0.5` значение должно считаться через эмпирический порог выборки, без специального аналитического случая.
13. Если эмпирический порог `v(q)` равен нулю или отрицателен, результатом для этого `q` и фильтра должна быть доля положительных значений в выборке.
14. CSV должен содержать 5000 строк данных, по одной строке на значение `q`.
15. Первый столбец CSV должен быть `q`.
16. Каждый столбец CSV после столбца `q` должен соответствовать одному фильтру.
17. Рекомендуемый заголовок CSV: `q,filter_0,filter_1,...,filter_N_minus_1`.
18. Порядок столбцов фильтров должен совпадать с порядком фильтров в `CIFAR_3x3.PCFilters.txt`.
19. Результаты должны быть воспроизводимыми при заданном seed; seed должен задаваться CLI-аргументом и иметь фиксированное значение по умолчанию.
20. Пути входных и выходного файлов, размер выборки, seed, параметр квантования `n` и параметры сетки `q` должны быть настраиваемы через CLI-аргументы, но значения по умолчанию должны соответствовать задаче.
21. При ошибках формата входных файлов, несовместимых размерах фильтров/изображений или невозможности вычислить значение функции скрипт должен завершаться с ненулевым кодом и понятным сообщением.
22. Значение `n` должно быть небольшим положительным целым числом; значение по умолчанию — `10`. Недопустимые значения `n` должны отклоняться при разборе CLI-аргументов.

## Нефункциональные требования

- Производительность: расчёт должен быть векторизован там, где это существенно; полный прогон для штатного числа фильтров не должен выполнять Python-циклы по всем `300000 * 5000` парам `sample, q`. Допустим отдельный цикл по небольшому числу уровней `n`, если он не доминирует по времени.
- Память / ресурсы: допустимо хранить массив изображений и одну выборку на текущий фильтр; не требуется хранить все выборки всех фильтров одновременно.
- Потокобезопасность / параллелизм: обязательный параллелизм не требуется; реализация не должна зависеть от изменяемого глобального состояния, кроме локального генератора случайных чисел.
- Совместимость (API, форматы файлов, версии toolchain): Python 3.8+; входные форматы должны оставаться совместимыми с текущими C++-читателями проекта.
- Наблюдаемость (логи, метрики): при обычном запуске выводить краткий прогресс по фильтрам и итоговый путь к CSV; подробное логирование не требуется.
- Прочее: CSV должен использовать стабильный десятичный формат чисел и разделитель `,`, чтобы файл открывался стандартными Python/Excel-инструментами.

## Открытые вопросы

Нет. Приняты решения:

- `q=0.5` считать через эмпирический порог выборки.
- При `v(q) <= 0` результатом считать долю положительных значений в выборке.
- CSV должен содержать первый столбец `q`.
- Выборки для разных фильтров независимы.
- Новые зависимости не фиксировать через `requirements.txt`.
- Эмпирический аналог должен использовать второй параметр `n`; значение по умолчанию — `10`.
- Вычисляемую функцию именовать `spar(S; q, n)`, где `S` — выборка или распределение.

## Критерии приёмки

### Автотесты

- [ ] Unit: парсер `*.PCFilters.txt` читает минимальный файл с несколькими фильтрами и сохраняет порядок фильтров.
- [ ] Unit: чтение raw-изображений отклоняет файл, размер которого не кратен `width * height * channels`.
- [ ] Unit: генератор плиток никогда не выбирает координаты, при которых фильтр выходит за границы изображения.
- [ ] Unit: свёртка на маленьком синтетическом изображении и фильтре совпадает с вручную рассчитанным скалярным произведением.
- [ ] Unit: эмпирическая функция на небольшой фиксированной выборке совпадает с заранее рассчитанными значениями для нескольких `q` и `n`.
- [ ] Unit: CLI отклоняет недопустимые значения `n` и использует `n=10` по умолчанию.
- [ ] Unit: при `v(q) <= 0` эмпирическая функция возвращает долю положительных значений в выборке.
- [ ] Integration/smoke: запуск на маленьком временном наборе изображений и фильтров создаёт CSV с заголовком, 5000 строками данных и ожидаемым числом столбцов.

- [ ] `python -m unittest` проходит для существующих и новых Python-тестов.
- [ ] Скрипт запускается из корня проекта с параметрами по умолчанию, если `Workplace/CIFAR10.bin` и `Workplace/CIFAR_3x3.PCFilters.txt` присутствуют.
