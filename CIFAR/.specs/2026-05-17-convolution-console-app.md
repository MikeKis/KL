# Консольное приложение convolution

- **Дата**: 2026-05-17
- **Статус**: Approved
- **Связанные issue/PR**: (нет)

## Цели

Реализовать консольное приложение `convolution`, принимающее один аргумент — путь к конфигурационному файлу, которое:

1. Загружает набор однотипных изображений из сырого бинарного файла в RAM.
2. Загружает один или несколько наборов фильтров в формате PCFilters (`LoadPCFilters`).
3. Для каждого изображения и каждого фильтра строит две неотрицательные карты свёртки (+фильтр и −фильтр, ReLU).
4. Для каждого фильтра вычисляет `Vmax` в режиме `def_sparsity` или `def_saturation` и проецирует значения свёрток в целые спайки `[0, 10]`.
5. Записывает бинарный файл значений свёрток, CSV со спайками и текстовый лог с метриками по фильтрам.

## Не цели

- Обучение или генерация фильтров (`PCFilters`, `pcfilCIFAR`).
- Потоковая обработка изображений, не помещающихся в RAM.
- GUI, сеть, интеграция с симулятором SNN (только подготовка данных).
- Поддержка форматов фильтров, отличных от PCFilters v1.
- Изменение формата файлов PCFilters.

## Функциональные требования

### Запуск

1. Синтаксис: `convolution <config.toml>` (единственный обязательный аргумент — путь к конфигу).
2. При отсутствии аргумента, ошибке чтения/разбора конфига или валидации — ненулевой код возврата и сообщение в `stderr`.

### Конфигурационный файл

Формат: **TOML 1.0** (текстовый, UTF-8, комментарии `#`, вложенные таблицы, массивы). Выбран за читаемость, явную типизацию и удобное задание списка «файл фильтров + stride» без дублирования ключей.

Пример:

```toml
[input]
file = "Workplace/CIFAR10.bin"
width = 32
height = 32

[[filters]]
file = "Workplace/PCFilters_3x3.txt"
stride = 1

[[filters]]
file = "Workplace/PCFilters_6x6.txt"
stride = 2

[projection]
mode = "def_sparsity"   # или "def_saturation"
criterion = 2.0         # смысл зависит от mode (см. ниже)

[output]
convolutions_bin = "Workplace/conv_values.bin"
spikes_csv = "Workplace/conv_spikes.csv"
log = "Workplace/convolution.log"
```

Обязательные поля:

| Поле | Соответствие п. ТЗ |
|------|-------------------|
| `input.file` | 1. Имя входного файла |
| `input.width` | 2. Ширина |
| `input.height` | 3. Высота |
| `filters[].file` | 4. Файлы фильтров (PCFilters) |
| `filters[].stride` | 5. Stride для каждого файла |
| `projection.mode` | 6. `def_sparsity` \| `def_saturation` |
| `projection.criterion` | 7. Критерий (`def_sparsity`: `0…10`; `def_saturation`: доля `0…1`) |
| `output.convolutions_bin` | 8. Бинарный выход |
| `output.spikes_csv` | 9. CSV спайков |
| `output.log` | 10. Лог |

### Входные изображения

1. Формат как в `pcfilCIFAR` / `unsfil/unsfilc.cpp`: подряд без заголовка, каждое изображение — `width × height × channels` байт `uint8`, порядок каналов **interleaved** (как OpenCV `CV_8UCn`, в т.ч. `CV_8UC3` для CIFAR10.bin проекта).
2. **`channels` не задаётся в конфиге** — берётся из метаданных PCFilters (`channels=` в каждом `[Filter*]`). Перед чтением изображений загружаются (или просматриваются) файлы фильтров; во всех `[[filters]]` значение `channels` должно совпадать, иначе ошибка.
3. Число изображений: `file_size / (width * height * channels)`; остаток от деления — ошибка.
4. Все изображения одного размера; тип матрицы `CV_8UC(channels)`.

### Валидация конфига

- `projection.mode` — ровно `def_sparsity` или `def_saturation`.
- `projection.criterion` при `def_sparsity`: `0 ≤ criterion ≤ 10`.
- `projection.criterion` при `def_saturation`: `0 < criterion ≤ 1` (доля).
- `filters` — непустой массив; `stride ≥ 1` для каждой записи.

### Фильтры и свёртка

1. Загрузка через существующую `LoadPCFilters(std::vector<cv::Mat> &, const char *)`.
2. Для каждой записи `[[filters]]` все фильтры из файла используют один `stride`.
3. Размер выхода для изображения `I` и фильтра `F` (как в `matConvolution` в `UnsupervisedFiltersMultiple.cpp`):

   `out_w = (I.cols - F.cols) / stride + 1`  
   `out_h = (I.rows - F.rows) / stride + 1`

   Целочисленное деление; если `out_w < 1` или `out_h < 1` — ошибка (фильтр не помещается).

4. Значение свёртки в позиции `(x, y)`: скалярное произведение патча изображения и фильтра (`cv::Mat::dot` на ROI, фильтр и патч приводятся к `CV_32F` при необходимости).
5. Для каждого фильтра `k` — **две** карты:
   - `conv_plus[k]` = `max(conv(I, +F_k), 0)`
   - `conv_minus[k]` = `max(conv(I, −F_k), 0)` (эквивалентно `max(−conv(I, +F_k), 0)` при той же формуле dot).
6. Порядок обхода для записи в выходные файлы (зафиксировать в реализации и в логе):
   - по индексу изображения (0 … N−1);
   - по порядку секций `[[filters]]` в конфиге;
   - по индексу фильтра внутри файла (0 … count−1);
   - полярность: `plus`, затем `minus`;
   - внутри карты: строка `y` от 0 до `out_h−1`, столбец `x` от 0 до `out_w−1`.

### Проекция в спайки

1. Период предъявления изображения фиксирован: **10** (используется только при интерпретации счётчиков спайков в CSV; на формулу `Vmax` не влияет).
2. Для каждого **физического фильтра** (одна матрица из PCFilters) один `Vmax > 0`, общий для обеих полярностей и всех изображений.
3. Проекция значения `v ≥ 0`:

   - если `v ≤ Vmax`: `spike = round(10 * v / Vmax)` с округлением к **ближайшему** целому (половинки — к чётному, banker's rounding как в C++ `std::round` / уточнить в реализации);
   - если `v > Vmax`: `spike = 10`.

   Итог: `spike ∈ {0, …, 10}`.

4. **def_sparsity**: подобрать `Vmax` так, чтобы **среднее** число спайков по **всем** значениям свёрток (обе полярности, все позиции, все изображения) для данного фильтра было **максимально близко** к `projection.criterion`. Допустимый диапазон критерия: **`0 ≤ criterion ≤ 10`** (среднее число спайков).
5. **def_saturation**: подобрать `Vmax` так, чтобы доля значений свёртки **строго больше** `Vmax` среди того же полного множества была **максимально близка** к `projection.criterion`. Критерий — **доля в интервале `(0, 1]`**, не проценты (например `0.05` означает 5%).
6. Алгоритм подбора `Vmax`: двухпроходный расчёт — накопить все `v`, затем **перебор только по уникальным значениям** из выборки (непрерывная оптимизация не требуется).
7. Если для фильтра **все** значения свёртки равны нулю (`max(V) = 0`) — **фатальная ошибка**: сообщение в `stderr`, ненулевой код возврата, **выходные файлы не создавать и не изменять**.

### Лог

Для каждого фильтра (с указанием файла и индекса в файле) записать строку:

| Режим | Поля в логе |
|-------|-------------|
| `def_sparsity` | `Vmax`, доля значений свёртки `> Vmax` |
| `def_saturation` | `Vmax`, достигнутое среднее число спайков по всем свёрткам с этим фильтром |

Дополнительно в начале лога: версия приложения, путь к конфигу, число изображений, режим, критерий, размеры карт.

### Выходные файлы

**Бинарный (`output.convolutions_bin`):**

- Все скалярные значения свёртки **после ReLU**, **до** проекции в спайки.
- Тип: `float64`, little-endian (native на x86_64).
- Порядок байт — как в п. «Порядок обхода».

**CSV (`output.spikes_csv`):**

- **Заголовка нет.**
- **Одна строка файла — одно изображение** (ровно столько строк, сколько изображений).
- В строке — все спайки `0`…`10` для этого изображения в порядке обхода из п. «Порядок обхода» (все `[[filters]]`, все фильтры, `plus`/`minus`, все позиции), разделитель полей — **запятая** (`,`), без пробелов (допустимы пробелы после запятой при чтении не требуются).
- Порядок значений в строке изображения `i` совпадает с подпоследовательностью бинарного файла, относящейся к изображению `i` (bin — плоская последовательность `float64`, CSV — та же последовательность, сгруппированная по строкам).

## Нефункциональные требования

- **Производительность**: полный проход по датасету в RAM; для CIFAR-10 (50k×32×32×3) и ~20 фильтров 3×3 целевое время — минуты на desktop CPU, без GPU.
- **Память**: хранить все изображения и все карты свёрток в памяти допустимо (явное требование ТЗ); оценка пиковой памяти логировать в debug-сборке опционально.
- **Потокобезопасность**: не требуется (однопоточное консольное приложение).
- **Совместимость**: C++17, OpenCV (как в корневом `CMakeLists.txt`), Windows (MSVC) и Linux (GCC/Clang) при наличии CMake-сборки.
- **Наблюдаемость**: основной отчёт — `output.log`; прогресс (опционально) — `stderr` при длительных прогонах.
- **Зависимости конфига**: парсер TOML (header-only, например [toml++](https://github.com/marzer/tomlplusplus) MIT) — единственная новая внешняя библиотека для этого таргета.

## Приложение: семантика режимов (формулы)

Пусть для фильтра `k` множество всех неотрицательных значений свёртки  
`V = { v_ij }` (обе полярности, все изображения и позиции), `N = |V|`.

**Проекция** при фиксированном `Vmax > 0`:

\[
\text{spike}(v) = \begin{cases}
10 & v > Vmax \\
\mathrm{round}(10 \cdot v / Vmax) & 0 \le v \le Vmax
\end{cases}
\]

**def_sparsity** — найти `Vmax`, минимизирующее `|mean_{v \in V} spike(v) - criterion|`.

**def_saturation** — найти `Vmax`, минимизирующее `|\#\{v \in V : v > Vmax\} / N - criterion|`.

**Лог:**

- `def_sparsity`: вывести найденный `Vmax` и `#{v > Vmax}/N`.
- `def_saturation`: вывести `Vmax` и `mean_{v \in V} spike(v)`.
