# Статическая библиотека PCFilters

- **Дата**: 2026-05-11
- **Статус**: Draft
- **Связанные issue/PR**: Нет

## Цели

Создать новую C++ статическую библиотеку `PCFilters` с кроссплатформенной сборкой в двух контурах:

1. Windows: проект Visual Studio 2022, подключённый к `CIFAR/CIFAR.sln`.
2. Linux: цель CMake, собираемая в существующей структуре проекта.

Реализовать и экспортировать три интерфейсные функции:

1. `void PCFilters(const std::vector<cv::Mat> &vmat_Images, std::vector<cv::Mat> &vmat_Filters, int FilterSize = 3, int SampleSize = 30000, int Seed = -1);`
2. `void SavePCFilters(const std::vector<cv::Mat> &vmat_Filters, const char *pchFile);`
3. `void LoadPCFilters(std::vector<cv::Mat> &vmat_Filters, const char *pchFile);`

Обеспечить вычисление PCA-фильтров по случайной выборке патчей `FilterSize x FilterSize x nch` из набора входных изображений (`nch` = 1 или 3), где количество главных компонент `n` равно `vmat_Filters.size()` на входе.

## Не цели

- Не менять публичный API существующей библиотеки `UnsupervisedFiltersMultiple`.
- Не добавлять в рамках этой задачи новые методы обучения фильтров, кроме PCA по заданному интерфейсу.
- Не реализовывать бинарный формат сохранения фильтров (только текстовый формат).
- Не покрывать дополнительные типы входных данных, кроме 2D изображений с 1/3 каналами.

## Функциональные требования

1. Библиотека `PCFilters` должна предоставлять заголовок с декларациями `PCFilters(...)`, `SavePCFilters(...)` и `LoadPCFilters(...)`.
2. `PCFilters(...)` должна валидировать вход:
   - `vmat_Images` не пустой;
   - все изображения имеют одинаковые `rows`, `cols`, количество каналов;
   - число каналов равно 1 или 3;
   - каждое изображение имеет `rows >= FilterSize` и `cols >= FilterSize`;
   - `FilterSize > 0`, `SampleSize > 0`;
   - `vmat_Filters.size() > 0`.
3. Из совокупности входных изображений должна формироваться случайная выборка из `SampleSize` патчей размера `FilterSize x FilterSize x nch`:
   - выборка **с повторениями**;
   - выбор координат патчей не выходит за границы исходных матриц.
4. Каждый патч должен разворачиваться в вектор размерности `D = FilterSize * FilterSize * nch`; далее по выборке вычисляются главные компоненты PCA.
5. Должны быть получены `n` компонент, где `n = vmat_Filters.size()` на входе; при `n > D` функция возвращает ошибку.
6. Результирующие векторы компонент должны быть упакованы в `cv::Mat`-тензоры фильтров формы `FilterSize x FilterSize` с исходным количеством каналов, типом `CV_32F`, и записаны в `vmat_Filters`.
7. `vmat_Filters` в `PCFilters(...)` используется как вход+выход: количество компонент `n` берётся из исходного `vmat_Filters.size()`, после чего содержимое вектора перезаписывается на месте результатом (без изменения `size()`).
8. При нарушении предусловий и/или некорректном формате входных данных функции должны выбрасывать исключения (`std::invalid_argument` для ошибок входа, `std::runtime_error` для ошибок обработки/парсинга).
9. В `PCFilters(...)` используется опциональный параметр `Seed` (`int Seed = -1`); если `Seed >= 0`, выборка детерминированная, если `Seed < 0` — недетерминированная.
10. Входные изображения в `PCFilters(...)` принимаются как `CV_8U`; для вычисления PCA допускается внутреннее преобразование в `CV_32F`.
11. `SavePCFilters(...)` должна сохранять построенные фильтры в текстовый файл в читаемом формате с достаточной информацией для последующей загрузки/проверки (индекс фильтра, размер, каналы, значения).
12. `LoadPCFilters(...)` должна читать файл, сохранённый `SavePCFilters(...)`, валидировать структуру данных и восстанавливать `vmat_Filters`.
13. Формат файла: секционный plain text (ini-like), удобный для ручной проверки и стабильного парсинга:
   - заголовок версии формата;
   - число фильтров;
   - для каждого фильтра: `rows`, `cols`, `channels`, `type`, затем список значений в порядке `row-major` с каналами в конце (HWC).
14. Должны быть добавлены цели сборки:
   - Visual Studio проект `PCFilters.vcxproj` (StaticLibrary, v143, Debug/Release, Win32/x64);
   - CMake-таргет `add_library(PCFilters STATIC ...)` с линковкой OpenCV.
15. Должен быть создан `CIFAR/CMakeLists.txt`, в котором подключается модуль `PCFilters` через `add_subdirectory(PCFilters)` (и при необходимости другие подпроекты `CIFAR`).
16. Проект `PCFilters` должен быть подключён в `CIFAR/CIFAR.sln`; на текущем этапе solution содержит только `PCFilters`.

## Нефункциональные требования

- Производительность: вычисление PCA для `SampleSize=30000` и типичного `FilterSize=3` не должно содержать лишних копирований и должно использовать эффективные операции OpenCV/линейной алгебры.
- Память / ресурсы: не допускать неконтролируемого роста памяти; хранить выборку в компактной матрице признаков `SampleSize x D`.
- Потокобезопасность / параллелизм: функции реентерабельны при отсутствии разделяемого изменяемого глобального состояния; генератор случайных чисел локален вызову и поддерживает опциональный `seed`.
- Совместимость (API, форматы файлов, версии toolchain): API в точности соответствует заявленным сигнатурам; сборка с VS2022 (toolset v143) и CMake >= 3.14 на Linux; совместимость с используемой в репозитории версией OpenCV.
- Наблюдаемость (логи, метрики): при ошибках входных параметров возвращать детализированные исключения/сообщения (например, `std::invalid_argument`) для диагностики.
- Прочее: формат текстового файла должен быть стабильным; сохранение/загрузка обратно совместимы в рамках версии формата.
